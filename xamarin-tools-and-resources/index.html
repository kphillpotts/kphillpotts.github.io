Design is a process

I think one of the most underrated parts of developing an application is the process of going from a sketch through wire-framing to mock-ups and prototyping. Next it‚Äôs on to the actual process.

Next it‚Äôs on to more of the Mockup and prototyping phases:

  • Zeplin ‚Äì Zeplin is a great way of getting the information about fonts and colors and stuff from those sketch files so you can use them in development.
  • Adobe Experience Designer (XD)¬† ‚Äì I‚Äôve been playing around with Adobe XD recently and I‚Äôve really enjoyed using this app. It may become my default designer tool (at least for mock-ups). Working with Data

Like it or not, a great deal of our time is working with Data. ¬†In a mobile context that mostly means hitting Web Services and local databases (SQLite or Realm). Here are my favorite tools and libraries:

  • Postman ‚Äì This is an essential tool for hitting web services checking requests and responses.   • Refit¬†‚Äì Simplify calling Rest services. ¬†If you want to simplify your code for calling web services give Refit a shot, you can basically just create an interface which describes your web service, then let it do all the hard work of accessing the service‚Ä¶ no need to worry about doing the serializing and deserializing yourself or creating GET and PUT methods‚Ä¶ it‚Äôll do it for you.
  • Polly¬†‚Äì Automatic retry policies.¬†The reality is that on mobile devices your connection sucks. It‚Äôs going to disappear for no apparent reason and sometimes the best thing is to just reissue a request. Polly is going to help you out here by allowing you to define automatic retry policies.
  • JSON.NET ‚Äì¬†The only library for parsing JSON results.

Rob Gibbens has a really great blog post on putting some of these tools together in his Resilient Network Services with Mobile Xamarin Apps article, you should definitely read this. http://arteksoftware.com/resilient-network-services-with-xamarin/

Debugging and Profiling

Let‚Äôs face it, unless you are super-human you probably spend half your time writing bugs. So debugging and profiling is an important part of #developerlife. Here are the tools I tend to use:

  • Xcode Instruments¬†‚Äì Event though it‚Äôs part of XCode and targetted at Objective-C and Swift apps, you can also use this to profile your Xamarin.IOS applications. Visual Design

I‚Äôm not too bad at laying out screens, but I still love to be able to get previews of what I‚Äôm doing. In Xamarin.IOS and Xamarin.Android you are fine because you have the Designers built into Visual Studio. For Xamarin.Forms things get a little bit trickier.   • Xamarin WorkBooks ‚Äì A great way to experimenting with different things and also useful for documentation. There is also the Xamarin Inspector which you can use to interrogate the UI of your application.
